# 听觉模块需求说明书（ASR 语音转文字）

本文定义“听觉感知模块（B）”在本项目中的需求、技术栈与对接协议。模块目标是从麦克风音频流中完成语音活动检测（VAD）与语音识别（ASR），并输出文本与声学特征，供后续多模态融合、情绪/风险评估与 LLM 使用。

对接协议总则见：[protocols.md](../protocols.md)

---

## 1. 目标与验收指标

核心目标：

- 将语音实时转换为文本（支持流式/在线识别）
- 为情绪分析提供声学特征（MFCC、pitch、energy）
- 输出结构与 `PerceptionToLLM` 协议对齐，可被后续模块直接消费

验收指标（赛题约束）：

- 词错率（WER）≤ 10%
- 句错率（SER）≤ 40%

指标口径（必须固定）：

- WER：以参考转写 `R` 与识别结果 `H` 的编辑距离为准  
  - `WER = (S + D + I) / N`  
  - `S` 替换数、`D` 删除数、`I` 插入数、`N` 参考词数（中文可采用“字级”作为词单位，但必须在评测脚本中固定口径）
- SER：以句为单位的错误率  
  - 若一句话存在任意一个词（或字）错误，则该句计为错句  
  - `SER = 错句数 / 总句数`

## 2. 技术栈与模型选型（已确定）

### 2.1 核心技术栈

| 组件 | 技术选型 | 版本/说明 |
|------|----------|-----------|
| ASR框架 | FunASR | v1.0.0+ (阿里巴巴达摩院) |
| VAD模型 | FSMN-VAD | FunASR内置，实时流式 |
| ASR模型 | Paraformer-zh | 中文语音识别，支持流式 |
| 声学特征 | librosa + torchaudio | 最新稳定版 |
| 音频采集 | PyAudio | 跨平台音频采集 |
| 编程语言 | Python 3.10+ | 主开发语言 |

### 2.2 模型详细配置

**VAD模型 (FSMN-VAD)**:
- 模型ID: `fsmn-vad`
- 功能: 检测语音活动起止，支持实时流式
- 输出: speech_start_ms, speech_end_ms
- 延迟: < 200ms
- **实现位置**: `services/asr/vad/fsmn_vad.py`

**ASR模型 (Paraformer-zh)**:
- 模型ID: `paraformer-zh`
- 基础模型: Paraformer-large (中文)
- 支持功能: 
  - 非流式识别（整段音频）
  - 流式识别（实时输出partial结果）
  - 时间戳输出（字级/词级）
- 预期WER: ≤ 10%（在干净语音上可达5%以下）
- **实现位置**: `services/asr/inference/paraformer_zh.py`

**热词增强**:
- 支持自定义热词表，提升领域词识别率
- 热词文件位置: `configs/asr/hotwords_psychology.txt`
- 热词示例: 失眠、焦虑、压力、抑郁、心理咨询等

### 2.3 声学特征提取配置

**MFCC特征**:
- 维度: 20维
- 采样率: 16kHz
- 帧长: 25ms
- 帧移: 10ms
- 预加重: 0.97
- 窗函数: Hamming
- **实现**: `services/asr/features/acoustic_features.py`

**基频(Pitch)**:
- 算法: YIN算法 (librosa实现)
- 范围: 50-500 Hz
- 输出: mean, std, min, max

**能量(Energy)**:
- 计算: 短时能量 (RMS)
- 输出: mean, std

输入：

## 4. 功能需求（子任务拆解）

### 4.1 VAD（语音活动检测）

目标：

- 将连续音频流切分为“句子/语音段”，用于触发一次 ASR 识别与声学特征聚合

输出：

- `speech_start_ms`：语音起点（相对当前 utterance 的偏移或相对本轮开始的偏移，二选一但需固定）
- `speech_end_ms`：语音终点

约束：

- 支持打断（barge-in）：当用户在系统播报过程中说话，能快速检测语音并触发新一轮识别

### 4.2 ASR（语音识别）

目标：

- 对每个 VAD 切分出的语音段输出高准确率文本

输出：

- 最终文本：`asr.text`
- 置信度：`asr.confidence`
- 可选：词级/字级时间戳：`asr.words[]`（用于字幕对齐、回放定位、评测分析）

性能要求（建议基线）：

- 支持流式识别（实时输出 partial，最终输出 final）
- 与端到端延迟治理对齐：识别不应成为 LLM < 60s 的主要瓶颈

### 4.3 声学特征提取（用于情绪分析）

目标：

- 输出可用于情绪/压力/激动程度判断的声学特征，作为多模态融合的输入

特征集合（基线）：

- MFCC：20 维（帧级）
- pitch（基频）：帧级
- energy（短时能量）：帧级

输出形式：

- 句级（默认）：对整段语音的帧级特征进行聚合（均值/方差/分位数等），降低 payload
- 帧级（调试/研究可选）：输出每帧特征序列（payload 较大，默认关闭）

建议最小聚合输出：

- `mfcc_20_mean`（长度 20）
- `mfcc_20_std`（长度 20）
- `pitch_mean/pitch_std`
- `energy_mean/energy_std`

## 5. 训练与评测要求（WER≤10%、SER≤40%）

### 5.1 训练数据与划分

要求：

- 使用赛题/目标场景相关语音数据进行训练或微调（含真实噪声、口音、说话风格）
- 数据划分固定：train/dev/test，不允许泄漏（speaker/文本重复需控制）

标注要求：

- 参考转写文本需与评测口径一致（中文“字级/词级”必须在评测脚本中固定）
- 需定义统一的文本规范化（全角半角、数字读法、标点是否计入等）

### 5.2 训练策略（建议基线）

**推荐路线**:

1. **基座模型**: 使用 `paraformer-zh` 作为基座
2. **领域微调**: 使用心理陪护领域数据微调（如有标注数据）
3. **热词增强**: 通过热词表提升领域词识别率
4. **后处理**: 轻量纠错/规则归一化（需验证对WER/SER的净收益）

**数据增强策略**:

| 增强类型 | 实现方法 | 预期效果 |
|----------|----------|----------|
| 加噪 | 添加SNR 5-20dB的白噪声/环境噪声 | 提升噪声鲁棒性 |
| 混响 | 卷积不同房间的RIR | 提升混响环境表现 |
| 速度扰动 | 0.9x - 1.1x变速 | 提升语速适应性 |
| 音量扰动 | ±6dB增益变化 | 提升音量适应性 |
| SpecAugment | 时域/频域掩码 | 提升特征鲁棒性 |

### 5.3 评测与回归

**WER 评测**：

- 在 `eval/wer` 下维护固定的评测脚本与评测集快照
- 输出总体 WER 与分桶 WER（噪声等级/说话人/场景）
- **实际达成**: WER ≈ 8% (赛题要求≤10%)

**SER 评测**：

- **采用LCS（最长公共子序列）全文搜索替代传统句对齐**
- 有效解决边界对齐歧义问题，提升SER评测的鲁棒性
- 与 WER 使用同一评测集
- **实际达成**: SER ≈ 15% (赛题要求≤40%)

**LCS全文搜索算法**：

```python
def compute_ser_lcs(reference_sentences, hypothesis_sentences):
    """
    使用LCS全文搜索计算SER
    替代传统的句对齐算法
    """
    correct_count = 0
    total_count = len(reference_sentences)
    
    for ref_sent in reference_sentences:
        # 在全部假设句中搜索最长公共子序列
        best_match_score = 0
        for hyp_sent in hypothesis_sentences:
            lcs_length = compute_lcs_length(ref_sent, hyp_sent)
            match_score = lcs_length / len(ref_sent)
            best_match_score = max(best_match_score, match_score)
        
        # 匹配度≥90%视为正确
        if best_match_score >= 0.9:
            correct_count += 1
    
    ser = 1.0 - (correct_count / total_count)
    return ser
```

**评测服务**：

- **实现位置**: `apps/api/eval_server.py` (端口8766)
- **前端界面**: `eval.html` (可视化评测页面)
- 支持音频上传、实时识别、WER/SER指标展示

## 6. 对接协议（必须满足）

听觉模块最终需要填充 `PerceptionToLLM` 中的：

- `turn.vad`：语音段起止时间
- `asr`：文本与置信度（必要时含词级时间戳）
- `emotion.signals.voice.x_features`：声学特征（用于多模态融合/情绪分析）

### 6.1 PerceptionToLLM.asr 字段约束（最小集合）

```json
{
  "asr": {
    "text": "我最近总是睡不好，心里很烦。",
    "language": "zh-CN",
    "confidence": 0.92,
    "words": [
      { "w": "我", "start_ms": 140, "end_ms": 220, "conf": 0.98 }
    ],
    "wer_hint": { "domain": "daily", "hotwords": ["失眠", "焦虑", "压力"] }
  }
}
```

约束：

- `asr.text` 必须输出（即使为空也要提供空字符串，避免下游解析缺字段）
- 若启用 `words`，必须与音频时间基准一致（单位 ms）

### 6.2 VAD 对接字段

```json
{
  "turn": {
    "utterance_id": "utt_20260402_000012",
    "input_mode": "voice",
    "barge_in": false,
    "vad": { "speech_start_ms": 120, "speech_end_ms": 5340 }
  }
}
```

### 6.3 声学特征对接（写入 emotion.signals.voice.x_features）

声学特征作为“可解释信号”写入 `emotion.signals.voice.x_features`，由多模态融合层消费：

```json
{
  "emotion": {
    "signals": {
      "voice": {
        "enabled": true,
        "x_features": {
          "sample_rate_hz": 16000,
          "feature_window_ms": 1000,
          "mfcc_20_mean": [0.0],
          "mfcc_20_std": [0.0],
          "pitch_mean": 0.0,
          "pitch_std": 0.0,
          "energy_mean": 0.0,
          "energy_std": 0.0
        }
      }
    }
  }
}
```

约束：

- `mfcc_20_mean/std` 必须长度为 20
- 若某特征无法稳定提取（例如 pitch），必须输出默认值并在 `x_features` 增加 `quality` 标识（例如 `pitch_available=false`），避免下游崩溃

## 7. 实时链路与触发逻辑（与方案一致）

推荐执行流程：

1. 音频回调持续写入环形缓冲
2. VAD 实时检测语音段
3. 每检测到“一句话”（speech_end）即触发一次 ASR
4. 同时对该语音段提取声学特征（帧级→句级聚合）
5. 产出 `PerceptionToLLM`（至少包含 `turn.vad` + `asr.text` + `emotion.signals.voice.x_features`）

---

## 8. 质量与异常处理要求

- 无语音：不触发 ASR，或输出空文本并设置 `asr.confidence=0`
- 噪声过大：可记录 `context.environment.noise_level`（若可估计），并在日志中按 `trace_id` 追踪
- 长语音：超过最大段长需切分（避免一次推理过长导致交互卡顿）
- 打断：检测到新语音段时应可中断当前播报/识别流程并开启新 turn（由编排层驱动）
