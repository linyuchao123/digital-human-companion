## 主要技术栈变更对照表

| 模块 | 原需求说明书描述 | 实际实现技术栈 | 变更原因 |
|------|------------------|----------------|----------|
| **全功能整合** | 支持视频上传+摄像头双模式 | 仅WebSocket实时对话模式 | 视频文件模式未实现 |
| **LLM引擎** | vLLM自部署 Qwen2.5-14B | Transformers本地加载 Qwen2.5-14B-Instruct (4-bit量化) | vLLM在Windows下兼容性问题 |
| **RAG向量库** | ChromaDB + BGE-large-zh | ChromaDB + **TF-IDF** (离线可用) | BGE需在线Embedding服务，改为TF-IDF实现离线降级 |
| **TTS语音合成** | Web Speech API | **CosyVoice流式TTS** (阿里云) | Web Speech中文效果差 |
| **数字人驱动** | 58维3D面部参数 | **25维情感特征** (15AU+2VA+8EXP) → 24个Live2D参数 | 官方验证集为25维 |
| **多模态融合** | BERT文本编码+MLP融合 | 简化版本（离线模式无BERT） | 降低复杂度，保障实时性 |
| **ASR评测** | 传统句对齐SER | **LCS全文搜索**替代句对齐 | 提升SER评测鲁棒性 |
| **前端界面** | 双栏布局 (45%/55%) | **三栏布局** (侧边栏+数字人+对话区) | 用户体验优化 |
| **用户系统** | 无 | **SQLite用户认证** + 历史对话管理 | 新增功能 |

### 6. 数字人面部行为驱动模型需求分析说明书
**需更新章节**：
- 3. 数据集：官方验证集25维 (15AU+2VA+8EXP)，非58维
- 4. 输出目标：25维情感特征，非3D面部参数
- 5. 模型架构：EmotionReactionTransformer (Transformer+CVAE)
- 6. 推理接口：25维→24个Live2D PARAM_*参数

**实际实现文件**：
- `digital_human_engine/train.jioaben/model.py` (EmotionReactionTransformer)
- `digital_human_engine/train.jioaben/emotion_to_live2d.py` (25→24映射)
- 权重文件：`digital_human_engine/checkpoints_v2/best_model.pt` (47MB)
- 评测脚本：`数字人面部行为驱动模型验证脚本/eval_emotion_metrics.py`

**模型架构**：
```
输入: speaker_emotion (25维) + audio (可选wav2vec)
  ↓
Transformer Encoder (4层×4头)
  ↓
CVAE生成头 (均值+对数方差→重参数化)
  ↓
输出: listener_emotion (25维: 15AU+2VA+8EXP)
  ↓
emotion_to_live2d.py映射
  ↓
24个Live2D PARAM_*参数
  ↓
WebSocket推送 → 前端驱动雫人物
```

**评测指标**：
- 训练: epoch=111, val_loss=0.4486
- 官方评测: FRCorr等指标（见`eval_emotion_metrics.py`）
