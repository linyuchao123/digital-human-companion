# 数字人面部行为驱动模型需求分析说明书

本文定义“模块五：数字人驱动与渲染”中最核心的“数字人面部行为驱动模型”的需求、训练数据、训练目标、推理接口与验收标准。该模型需要训练，训练数据集路径为：`d:\服创比赛\train`。

对接协议总则见：[protocols.md](../protocols.md)

---

## 1. 目标与验收要求

### 1.1 目标

- 提供不少于 2 个表现力丰富的 2D/3D 数字人形象
- 数字人形象由“面部行为驱动模型”驱动，具备：
  - 自然流畅的语音对话表现（与 TTS 输出同步）
  - 精准口型同步（Lip-sync）
  - 丰富面部表情（与对话情绪一致且可控）
  - 适度肢体动作（点头/轻微手势等，避免夸张）
- 交互自然、低延迟，支持实时播放（建议 25fps 以上）

### 1.2 关键技术指标（建议基线）

- 动画输出帧率：25 fps（与训练数据对齐）
- 口型同步延迟：音频开始播放后 200ms 内出现正确口型趋势
- 表情响应延迟：LLM 输出情绪/表达意图后 500ms 内可见表情变化
- 动作稳定性：避免抖动（提供平滑/去噪），无明显跳变

## 2. 系统位置与对接关系

本模型位于 `services/avatar`（Render/Drive）内部，负责把“语音信号 + 情绪意图”转换为“逐帧面部参数（及可选肢体动作）”，供 2D/3D 渲染引擎实时驱动。

上游输入来源：

- `services/tts`：语音波形（或流式音频 chunk）
- `services/llm`：`LLMToDriver.render.avatar`（表情/动作意图）与 `LLMToDriver.render.voice.emotion`（情绪强度）
- `services/multimodal_fusion`（可选）：`PerceptionToLLM.emotion`（用户情绪与风险，影响数字人表达策略）

下游输出去向：

- 3D 渲染（Unity / Unreal / MetaHuman）
- 2D 渲染（Live2D + SDK）
- 客户端播放器（Web/Native）：接收逐帧参数并驱动播放

协议对齐原则：

- LLM 输出保持“高层意图”（如 `expression.name/intensity`、`gesture.name/intensity`）
- 面部行为驱动模型输出“低层参数”（blendshape/3D 参数/viseme），不要求 LLM 直接输出每个 blendshape

## 4. 输出目标定义（训练标签与评测格式）

### 4.1 训练监督信号

**主监督 - 58维面部参数**:

- 来源: `3D_FV_files/**/*.npy`
- 格式: shape=(T, 1, 58)
- 帧率: 25fps
- 说明: 58维参数语义以数据集官方定义为准（包含嘴部/表情/姿态等）

**辅助监督 - 25维情绪特征（强烈建议）**:

- 来源: `Emotion/**/*.csv`
- 格式: shape=(T, 25)
- 帧率: 25fps
- 维度分解:
  - 15AU: `AU1, AU2, AU4, AU6, AU7, AU9, AU10, AU12, AU14, AU15, AU17, AU23, AU24, AU25, AU26`
  - 2VA: `valence, arousal`
  - 8EXP: `Neutral, Happy, Sad, Surprise, Fear, Disgust, Anger, Contempt`

**多任务训练收益**:
- 让模型更容易"按情绪意图调表情"
- 便于将LLM输出的情绪条件作为控制量注入
- 辅助监督有助于提升主任务的泛化能力

### 4.2 评测输出格式（必须遵循）

**评测文件**: `prediction_emotion.npy`

**文件形状**: `[N, K, T, 25]`

| 维度 | 含义 | 值 |
|------|------|-----|
| N | 验证样本数 | 由person_specific_val.csv展开后确定 |
| K | 每样本候选序列数 | 默认 **10** |
| T | 序列长度 | 默认 **750** (30秒@25fps) |
| 25 | 面部行为特征维度 | 15AU + 2VA + 8EXP |

**25维特征顺序（严格固定）**:

```python
feature_order = [
    # 15 AU (0-14)
    "AU1", "AU2", "AU4", "AU6", "AU7", "AU9", "AU10", 
    "AU12", "AU14", "AU15", "AU17", "AU23", "AU24", "AU25", "AU26",
    # 2 VA (15-16)
    "valence", "arousal",
    # 8 Expression (17-24)
    "Neutral", "Happy", "Sad", "Surprise", "Fear", "Disgust", "Anger", "Contempt"
]
```

**重要约束**:

1. 样本顺序必须严格遵循 `person_specific_val.csv` 的展开顺序
2. 每样本生成 **K=10** 条候选序列
3. 序列长度固定为 **T=750** 帧
4. 特征维度严格为 **25维**，顺序不可更改
5. 数值范围建议归一化到 [0, 1] 或 [-1, 1]

### 4.3 内部表示与评测格式转换

模型内部可以使用任意表示（如58D面部参数），但在评测前必须转换为25维情绪特征格式：

```python
# 转换流程示例
internal_output = model(audio)  # shape: (T, 58)

# 方式1: 如果58D可直接映射到25D
emotion_25d = mapping_58d_to_25d(internal_output)  # shape: (T, 25)

# 方式2: 通过辅助任务头直接输出25D
emotion_25d = model.emotion_head(internal_features)  # shape: (T, 25)

# 扩展为10个候选（多样性生成）
prediction = np.stack([emotion_25d + noise_i for i in range(10)], axis=0)  # (10, T, 25)

# 保存为评测格式
np.save("prediction_emotion.npy", prediction)  # 最终shape: (N, 10, 750, 25)
```

## 5. 模型功能需求

### 5.1 口型同步（Lip-sync）

需求：

- 输入 TTS 音频，输出与音素变化一致的嘴部开合/圆唇/咬唇等口型参数变化

实现路线（推荐：参数驱动，非逐帧合成视频）：

- 直接训练 Audio→58D 参数序列（嘴部维度在 58D 中体现）
- 或在 58D 基础上增加“viseme 层”（将音频映射到 K 个口型基元，再映射到引擎参数）

评测建议：

- 嘴部相关参数与 GT 的 MSE/相关系数
- 若使用视频对齐：可额外计算 lip landmark error（需要提取嘴部关键点）

### 5.2 表情驱动（情绪可控）

需求：

- 能将 LLM 输出的情绪意图（例如 `expression.name/intensity`、`voice.emotion.primary/intensity`）映射为面部参数变化
- 表情变化应与口型同时存在，不互相“抢控制权”

实现路线：

- 条件生成：Audio 编码器 + Emotion/Expression 条件向量 → 解码器输出 58D
- 条件来源：
  - LLM 输出：`LLMToDriver.render.avatar.expression` 与 `LLMToDriver.render.voice.emotion`
  - 融合输出：`PerceptionToLLM.emotion` 或 `x_ext.fusion.emotion_probs/va`

### 5.3 适度肢体动作（可选增强）

需求：

- 输出轻量动作指令（点头、轻微摆头、手势），与语气/情绪一致

实现路线：

- 规则/状态机优先：从 `LLMToDriver.render.avatar.gesture` 触发预设动作
- 模型预测可作为增强（需额外动作标签与数据集支持；当前数据集主要提供面部参数，肢体建议先走规则预设）

### 6.3 网络结构（已确定）

**EmotionReactionTransformer架构（实际实现）**:

```
输入: speaker_emotion (25维: 15AU+2VA+8EXP)
         ↓
┌─────────────────────────────────────┐
│  Transformer Encoder                 │
│  - 4层×4头注意力机制                │
│  - d_model=256                      │
│  - position encoding                │
└─────────────────────────────────────┘
         ↓
    (T, 256) 时序编码特征
         ↓
┌─────────────────────────────────────┐
│  CVAE生成头                         │
│  - FC(256→256) mu                   │
│  - FC(256→256) logvar               │
│  - 重参数化: z = mu + eps*std       │
│  - FC(256→25) 输出层                │
└─────────────────────────────────────┘
         ↓
    输出: listener_emotion (25维: 15AU+2VA+8EXP)
         ↓
┌─────────────────────────────────────┐
│  25维→24个Live2D参数映射             │
│  - 规则映射函数                     │
│  - PARAM_*格式输出                  │
└─────────────────────────────────────┘
         ↓
    24个PARAM_*参数驱动Live2D
```

**网络配置详情**:

| 组件 | 配置 | 参数量 |
|------|------|--------|
| Transformer Encoder | 4层×4头, d_model=256 | ~2.5M |
| CVAE生成头 | mu+logvar+decoder | ~200K |
| 25D→24PARAM映射 | 规则函数 | 0 |
| 总计 | - | ~2.7M |

### 6.4 损失函数（已确定）

**损失函数（实际实现）**:

```python
# EmotionReactionTransformer损失

# 各损失定义
```

**25维情感特征 → 24个PARAM_*参数映射规则**:

```python
def emotion_25d_to_live2d_params(emotion_25d):
    """
    25维情感特征 → 24个Live2D PARAM_*参数
    emotion_25d: [15AU, valence, arousal, 8EXP]
    """
    AU = {name: emotion_25d[i] for i, name in enumerate([
        'AU1','AU2','AU4','AU6','AU7','AU9','AU10','AU12',
        'AU14','AU15','AU17','AU23','AU24','AU25','AU26'
    ])}
    valence = emotion_25d[15]   # -1~1, 负=悲伤, 正=开心
    arousal = emotion_25d[16]   # -1~1, 负=平静, 正=激动
    exp_probs = emotion_25d[17:25]  # 8类表情概率

    return {
        # 头部姿态
        "PARAM_ANGLE_X": arousal * 25.0,
        "PARAM_ANGLE_Y": (AU['AU1'] - AU['AU4']) * 15.0,
        "PARAM_ANGLE_Z": valence * 10.0,
        # 眼睛
        "PARAM_EYE_L_OPEN": 1.0 - AU['AU6'] * 0.5 - AU['AU7'] * 0.5,
        "PARAM_EYE_R_OPEN": 1.0 - AU['AU6'] * 0.5 - AU['AU7'] * 0.5,
        "PARAM_EYE_BALL_X": arousal * 0.3,
        "PARAM_EYE_BALL_Y": valence * 0.2,
        # 眉毛
        "PARAM_BROW_L_FORM": AU['AU1'] + AU['AU2'] - AU['AU4'],
        "PARAM_BROW_R_FORM": AU['AU1'] + AU['AU2'] - AU['AU4'],
        "PARAM_BROW_L_ANGLE": -AU['AU4'],
        "PARAM_BROW_R_ANGLE": -AU['AU4'],
        # 嘴巴
        "PARAM_MOUTH_OPEN_Y": AU['AU25'] + AU['AU26'],
        "PARAM_MOUTH_FORM": valence,
        "PARAM_MOUTH_SIZE": AU['AU12'],
        "PARAM_MOUTH_SHRUG": AU['AU17'],
        # 脸颊
        "PARAM_CHEEK": exp_probs[1],  # Happy概率
        # 身体
        "PARAM_BODY_ANGLE_X": arousal * 10.0,
        "PARAM_BODY_ANGLE_Y": valence * 5.0,
        "PARAM_BODY_ANGLE_Z": arousal * 5.0,
    }
```

## 11. 评测指标（官方定义）

### 11.1 官方评测指标

根据官方评测脚本 `eval_emotion_metrics.py`，评测指标包括：

| 指标 | 说明 | 优化方向 | 权重 |
|------|------|----------|------|
| **FRCorr** | 与邻居集合的最大CCC相关系数 | 越大越好 | 核心指标 |
| **FRCorr*** | 个性化FRCorr（与FRCorr相同） | 越大越好 | 核心指标 |
| **FRdist** | 加权DTW距离 | 越小越好 | 核心指标 |
| **FRDiv** | 候选序列内部多样性 | 适中 | 辅助指标 |
| **FRDvs** | 跨样本差异 | 适中 | 辅助指标 |
| **FRVar** | 时序变化强度 | 适中 | 辅助指标 |
| **FRSyn** | 音画同步偏移 | 越小越好 | 核心指标 |
| **FRRea** | 真实性（需渲染+FID） | 越大越好 | 组委会复评 |

### 11.2 核心指标详解

**FRCorr / FRCorr***:

```
score(n, k) = max_{j in A(n)} CCC(pred[n, k], gt[j])
FRCorr = (1 / N) * sum_n sum_k score(n, k)
```

- CCC: Concordance Correlation Coefficient（一致性相关系数）
- A(n): 邻居集合，由官方邻居矩阵定义
- 计算时分别对AU/VA/EXP加权

**FRdist**:

```
dist(n, k) = min_{j in A(n)} d(pred[n, k], gt[j])
d(x, y) = (1/15)*DTW(AU) + 1*DTW(VA) + (1/8)*DTW(EXP)
```

- DTW: Dynamic Time Warping（动态时间规整）
- VA部分权重最高（为1），说明VA对齐最重要

**FRSyn**:

```
FRSyn = mean(|offset(n, k)|)
```

- offset: 在±2fps范围内计算时滞互相关的峰值偏移
- 衡量音画同步性，越小表示同步越好
