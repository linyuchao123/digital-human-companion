# 多模态融合与心理状态理解模块需求分析说明书

本文定义“模块三：多模态融合与心理状态理解（C 层）”的需求、输入输出、融合对齐策略与对接方式。模块目标是整合视觉与听觉（以及文本）信号，在统一时间基准下完成对齐与融合，输出可直接用于后续大语言模型（LLM）推理的“结构化情绪/心理状态信号”。

对接协议总则见：[protocols.md](../protocols.md)

---

## 1. 目标与范围

核心目标：

- 对齐：把视频帧、音频特征与 ASR 文本按时间窗对齐，得到对齐后的多模态序列或片段级特征
- 融合：将文本（语义）、语音（声学）与视觉（表情/AU/姿态/视线）融合为一个稳定的融合向量（fused embedding）
- 理解：产出可解释的情绪状态（分类 + VA）与心理风险评估结果（用于陪护策略与安全升级）
- 对接：将输出以稳定字段写入 `PerceptionToLLM`，便于 LLM 直接消费，不引入维度漂移

范围内：

- 时空对齐（时间戳同步、插值/聚合）
- 特征融合（基线拼接 + MLP，增强 Cross-Attention/Transformer）
- 情绪分类与 VA 回归
- 心理风险评估（筛查式分数/风险等级输出）
- 输出协议与降级策略

范围外（由其他模块负责）：

- ASR 模型训练与推理（见《听觉模块需求说明书》）
- 视觉面部特征提取（见《视觉模块需求说明书》）
- LLM 对话生成与工具调用

---

## 2. 输入数据（来自视觉与听觉模块）

本模块消费 `PerceptionToLLM` 中的多模态原始/半结构化输入（由感知层提供）。

### 2.1 听觉输入（B）

必选：

- `asr.text`：语音识别结果（作为文本模态输入）
- `turn.vad`：语音段起止（用于对齐窗口与句级聚合）

可选（推荐）：

- `emotion.signals.voice.x_features`：声学特征（MFCC20、pitch、energy 的句级聚合或帧级序列）

### 2.2 视觉输入（A）

可选但推荐（若用户开启摄像头）：

- `vision.features.face.au_15`：与数据集对齐的 15AU（固定集合：AU1, AU2, AU4, AU6, AU7, AU9, AU10, AU12, AU14, AU15, AU17, AU23, AU24, AU25, AU26）
- `vision.summary.va`：视觉侧估计 VA（valence/arousal）
- `vision.summary.expression_8`：8 类表情概率（Neutral/Happy/Sad/Surprise/Fear/Disgust/Anger/Contempt）
- `vision.features.face.head_pose`：pitch/yaw/roll
- `vision.features.face.gaze`：视线向量（可选）

备注：

- `landmarks_478` 与 `blendshapes_52` 默认不用于融合（payload 大），仅用于调试/研究或需要更细驱动时开启

## 3. 时间基准与时空对齐（必做）

### 3.1 时间基准

统一时间基准为 `epoch_ms`。

- 音频块、VAD 边界、视觉帧都必须带时间戳
- 以当前 turn 的起点作为相对时间 0ms 亦可，但必须明确并保持一致

### 3.2 对齐窗口

默认对齐窗口：

- `window_ms = 1000`（1s 滑窗）
- `hop_ms = 200`（可配置，推荐 200~500ms）

句级对齐（与 VAD 关联）：

- 使用 `turn.vad.speech_start_ms/speech_end_ms` 定义一句话的有效范围
- 在该范围内对视觉与声学特征做聚合（均值/方差/分位数）

### 3.3 插值与聚合策略

视觉帧 → 窗口特征：

- 默认对齐到 15 fps，窗口内做均值/方差聚合
- head_pose/gaze 可做均值 + 稳定性指标（方差）

声学帧 → 窗口特征：

- 若输入为帧级序列：按同一窗口聚合
- 若输入已是句级聚合：可直接使用并标记 `feature_window_ms`

文本对齐：

- 文本无天然帧率，默认以“整句”为单位生成 embedding，并复制到该句对应窗口
- 若需要更精细：可按 ASR 的 `words[]` 时间戳对齐到窗口（增强项）

### 4.1 核心技术栈

| 组件 | 技术选型 | 版本/说明 |
|------|----------|-----------|
| 深度学习框架 | PyTorch 2.0+ | 主框架 |

### 4.2 文本特征（Text）

**实际实现（已降级）**:

- ~~模型: `bert-base-chinese` (HuggingFace Transformers)~~
- ~~输出维度: 768维~~

**TF-IDF文本特征流程**:

```python
from sklearn.feature_extraction.text import TfidfVectorizer

# TF-IDF降级实现
vectorizer = TfidfVectorizer(
    max_features=500,
    stop_words=None,  # 自定义中文停用词
    ngram_range=(1, 2)
)
# 输出: 500维稀疏向量 -> MLP融合
```

**合并实现 (BERT离线模式 - 如已安装模型)**:

```python
from transformers import BertTokenizer, BertModel
import torch

class BertTextEncoder:
    def __init__(self, model_path):
        self.tokenizer = BertTokenizer.from_pretrained(
            model_path, local_files_only=True  # 离线模式
        )
        self.model = BertModel.from_pretrained(
            model_path, local_files_only=True
        )
    
    def encode(self, text: str) -> torch.Tensor:
        inputs = self.tokenizer(text, return_tensors="pt", max_length=128)
        with torch.no_grad():
            outputs = self.model(**inputs)
        return outputs.last_hidden_state[:, 0, :]  # [CLS] 768维
```

## 5. 融合方法与输出任务

### 5.1 融合方法（实际实现）

> **实际实现说明**: 当前版本采用规则基线融合，尚未实现MLP深度融合网络。

**实际实现：规则基线融合**

```python
class FusionEngine:
    """多模态融合引擎 - 实际实现为规则基线版本"""
    
    def fuse(self, vision_features, audio_features, text_features):
        """
        规则基线融合策略:
        1. 视觉特征直接传递到LLM上下文
        2. 音频特征直接传递
        3. 文本由LLM直接处理
        4. 无跳模态注意力融合
        """
        emotion_state = self._estimate_emotion(
            vision_features, audio_features
        )
        return {
            "vision": vision_features,
            "audio": audio_features,
            "text": text_features,
            "emotion": emotion_state,
            "fused_embedding": None  # 深度融合待后续优化
        }
    
    def _estimate_emotion(self, vision, audio):
        """规则基线：优先视觉特征估计情绪"""
        if vision and vision.get("va"):
            return vision["va"]  # 直接使用视觉侧VA
        elif audio and audio.get("emotion"):
            return audio["emotion"]  # 回退到音频侧
        else:
            return {"valence": 0.0, "arousal": 0.0}  # 默认中性
```

**实现位置**: `services/multimodal_fusion/fusion_engine.py`

**待优化方案A: MLP融合网络**

```
输入特征拼接:
┌─────────────────────────────────────────────────────────────┐
│  文本特征 (768D) │ 语音特征 (44D) │ 视觉特征 (25D)          │
└─────────────────────────────────────────────────────────────┘
                         ↓
                   特征拼接 (837D)
                         ↓
              ┌──────────────────────┐
              │  MLP融合网络         │
              │  - FC(837→512)       │
              │  - ReLU + Dropout    │
              │  - FC(512→256)       │
              │  - ReLU              │
              │  - FC(256→128)       │
              └──────────────────────┘
                         ↓
              融合向量 (128D) → 下游任务
```

**网络配置**:
- 输入维度: 837维 (768+44+25)
- 隐藏层: [512, 256]
- 输出维度: 128维 (融合向量)
- 激活函数: ReLU
- Dropout: 0.3
- **实现位置**: `services/multimodal_fusion/models/mlp_fusion.py`

**方案B: Transformer融合（增强方案）**

```
多模态Token输入:
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│ [TEXT]      │  │ [AUDIO]     │  │ [VISION]    │
│ 768D        │  │ 44D→128D    │  │ 25D→128D    │
│ (投影到128) │  │ (投影到128) │  │ (投影到128) │
└─────────────┘  └─────────────┘  └─────────────┘
       ↓                ↓                ↓
       └────────────────┼────────────────┘
                        ↓
              ┌──────────────────────┐
              │  Transformer Encoder │
              │  - 3 layers          │
              │  - 4 heads           │
              │  - d_model=128       │
              └──────────────────────┘
                        ↓
              融合向量 (128D)
```

**性能对比**:

| 方案 | 参数量 | 推理延迟 | 实现难度 | 推荐场景 |
|------|--------|----------|----------|----------|
| MLP | ~500K | < 10ms | 低 | 基线/快速落地 |
| Transformer | ~2M | < 30ms | 中 | 增强/精度优先 |

### 5.2 输出任务

1) 融合向量输出：

- 输出固定长度 `fused_embedding`（用于下游策略或作为 LLM 附加上下文信号）

2) 情绪分类：

- 输出表情/情绪概率分布
- 数据集表情为 8 类：Neutral/Happy/Sad/Surprise/Fear/Disgust/Anger/Contempt（以此为主口径）
- 如下游只需要 7 类：可通过映射把 Contempt 合并到 Neutral 或 Anger（必须固定映射规则并写入配置）

3) VA 回归：

- 输出 `valence/arousal`（范围建议 [-1, 1]）

4) 心理风险评估（筛查式）：

- 输出 PHQ-9/GAD-7 的“模型估计分数”与置信度（不是临床诊断）
- 输出风险等级 `low/medium/high` 供 LLM 策略与升级使用

## 6. 对接输出（写入 PerceptionToLLM，方便 LLM 调用）

本模块的输出不新增强制顶层字段，默认写入以下两处：

- `emotion`：融合后的情绪结论（LLM 直接使用）
- `x_ext`：融合细节与心理状态结构化输出（允许扩展，不影响解析）

### 6.1 emotion 字段写入规则（必须）

- `emotion.primary`：取概率最高的类别或规则融合后的主情绪
- `emotion.valence/arousal`：来自融合回归结果（优先融合输出，其次单模态）
- `emotion.confidence`：融合输出的整体置信度（0~1）
- `emotion.signals.*.enabled`：标记哪些模态参与了本轮融合

### 6.2 x_ext.fusion（建议）

`x_ext.fusion` 用于给 LLM 提供可解释、可复现的结构化状态（以及给评测/回放使用）。

```json
{
  "x_ext": {
    "fusion": {
      "version": "fusion_v1",
      "window_ms": 1000,
      "hop_ms": 200,
      "modalities": { "text": true, "voice": true, "vision": true },
      "fused_embedding": {
        "dim": 256,
        "values": [0.0]
      },
      "emotion_probs": {
        "Neutral": 0.22,
        "Happy": 0.02,
        "Sad": 0.58,
        "Surprise": 0.04,
        "Fear": 0.03,
        "Disgust": 0.03,
        "Anger": 0.04,
        "Contempt": 0.04
      },
      "va": { "valence": -0.3, "arousal": 0.25, "confidence": 0.6 },
      "psych_state": {
        "phq9_score_est": 9,
        "gad7_score_est": 7,
        "risk_level": "medium",
        "confidence": 0.55
      },
      "alignment": {
        "time_base": "epoch_ms",
        "vad": { "speech_start_ms": 120, "speech_end_ms": 5340 },
        "vision_frames_used": 14,
        "audio_frames_used": 98
      }
    }
  }
}
```

约束：

- `emotion_probs` 的 key 必须来自数据集表情集合（8 类）并保持大小写一致
- `fused_embedding.values` 允许省略（若 payload 受限），但必须保留 `dim` 与版本号，便于线上/离线一致
- 若某模态不可用（无摄像头/无声学特征）：`modalities.*=false`，并在融合时自动降级

## 7. 降级策略与缺失数据处理（必须）

视觉缺失（摄像头关闭/无脸）：

- 退化为“文本 + 语音”融合
- `emotion.signals.vision.enabled=false`，并保留 `x_ext.fusion.modalities.vision=false`

语音特征缺失（仅文本输入）：

- 退化为“文本 + 视觉”或“仅文本”

ASR 为空或置信度极低：

- 不得输出不可信文本；可将 `asr.text` 置空并降低 `emotion.confidence`
- 本模块仍可利用语音/视觉信号输出“情绪强度/风险等级”供 LLM 使用

---

## 8. 性能与资源约束

建议基线（以不影响端到端交互为准）：

- 融合推理（每 turn）：尽量控制在 50~200ms 量级
- 窗口聚合：在线增量计算，避免每轮全量回放
- 输出限流：视觉按 15 fps 聚合；声学默认句级聚合；文本按句级 embedding

---

## 9. 训练、评测与可追踪性（建议）

训练数据对齐：

- 视觉 AU/表情类别必须与数据集列严格一致（见《视觉模块需求说明书》）
- 文本口径与 ASR 评测口径一致（标点/数字规范化固定）

评测指标（建议）：

- 情绪分类：Accuracy / Macro-F1
- VA 回归：CCC / MAE
- 风险/分数回归：MAE / 分桶准确率（low/medium/high）

可追踪性：

- 全流程必须携带 `trace_id/session_id/turn_id`
- 融合模块需在日志中记录：参与模态、窗口参数、输出置信度与风险等级（不得记录敏感原始图像/音频内容）
