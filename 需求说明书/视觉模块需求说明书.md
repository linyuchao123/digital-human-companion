# 视觉模块需求说明书（MediaPipe）

本文定义“视觉感知模块（A）”在本项目中的需求、技术选型与对接协议。模块目标是从摄像头视频流实时提取面部特征，并输出给“多模态融合/LLM”与“数字人驱动”使用。

对接协议总则见：[protocols.md](../protocols.md)

---

## 1. 背景与目标

视觉模块在系统总体架构中属于“数据处理与感知层（A）”，用于提供与用户状态相关的客观信号，主要支撑：

- 情绪/心理状态评估：为 `valence/arousal` 与风险识别提供依据
- 交互增强：视线/点头等信号用于判断关注度与对话时机
- 数字人驱动对齐：在需要时可直接使用 blendshape 做面部驱动或表情提示

核心目标：

- 实时：15~30 fps（建议下游使用 15 fps 限流）
- 低延迟：LIVE_STREAM 异步模式，保证交互自然
- 可对接：输出结构必须可直接嵌入 `PerceptionToLLM.vision`

## 2. 技术栈与模型（已确定）

### 2.1 核心技术栈

| 组件 | 技术选型 | 版本/说明 |
|------|----------|-----------|
| 视觉推理框架 | MediaPipe Tasks Vision | **v0.10.x** |
| 核心模型 | Face Landmarker | 带 blendshape 版本 |
| 编程语言 | Python 3.10+ | 主开发语言 |
| 关键依赖 | opencv-python, mediapipe, numpy | 最新稳定版 |
| 姿态解算 | OpenCV solvePnP | 内置模块 |
| 平滑滤波 | 自定义 IIR 实现 | 一阶/二阶可选 |
| **实现位置** | **`services/vision/inference/mediapipe_face.py`** | **MediaPipeFaceInference类** |

### 2.2 模型文件

- **主模型**: `face_landmarker.task`（要求带 blendshape 支持）
  - 下载地址: [MediaPipe Model Garden](https://developers.google.com/mediapipe/solutions/vision/face_landmarker)
  - 模型大小: ~30MB
  - 输出: 478 landmarks + 52 blendshapes

### 2.3 运行模式

- `LIVE_STREAM`（异步）用于摄像头实时流
- 支持 `VIDEO` 模式用于离线视频处理（评测/调试）

### 2.4 硬件要求

- **最低配置**: CPU (Intel i5/AMD Ryzen 5 及以上), 8GB RAM
- **推荐配置**: 带 GPU 加速 (CUDA 可选，MediaPipe 主要使用 CPU)
- **摄像头**: 720p@30fps 及以上，支持自动对焦

## 4. 子任务拆解（与方案图片一致）

### 4.1 面部关键点检测

技术选型：

- MediaPipe Face Landmarker（带 blendshape）
- **API版本**: v0.10.x (使用 `mp.Image` 而非旧版 `MPImage`)

输出：

- 478 个关键点（landmarks）
- 52 维 blendshape（建议使用 MediaPipe 输出的命名系数）

备注：

- 推荐使用带 blendshape 的模型，便于与数字人表情/口型控制对齐
- **v0.10.x API变更**: `mp.tasks.vision.MPImage` → `mp.Image`
- **模型路径**: 必须使用纯英文路径（如 `d:/face_landmarker.task`）
- **测试验证**: 16/16 协议验证通过

### 4.2 AU 强度估计（15 维）

技术路线：

- 通过 blendshape 映射为 AU 强度

映射工具/表：

- `NAME2AUWEIGHT` 映射表（将多个 blendshape 加权汇总到 AU）
- **映射表定义位置**: `services/vision/config/au_mapping_v1.json`

**15维AU定义（与数据集严格对齐）**:

| AU编号 | 名称 | 对应面部动作 | 相关Blendshape |
|--------|------|--------------|----------------|
| AU1 | 内侧眉毛提升 | 抬眉（内侧） | browInnerUp |
| AU2 | 外侧眉毛提升 | 抬眉（外侧） | browOuterUpLeft/Right |
| AU4 | 眉毛降低 | 皱眉 | browDownLeft/Right |
| AU6 | 脸颊提升 | 眯眼/笑眼 | cheekSquintLeft/Right |
| AU7 | 眼睑收紧 | 眨眼/眯眼 | eyeSquintLeft/Right |
| AU9 | 鼻子皱起 | 皱鼻 | noseSneerLeft/Right |
| AU10 | 上唇提升 | 上唇抬起 | mouthUpperUpLeft/Right |
| AU12 | 嘴角拉动 | 微笑 | mouthSmileLeft/Right |
| AU14 | 嘴角下压 | 抿嘴 | mouthDimpleLeft/Right |
| AU15 | 嘴角下垂 | 悲伤 | mouthFrownLeft/Right |
| AU17 | 下巴提升 | 抬下巴 | mouthShrugLower |
| AU23 | 嘴唇收紧 | 抿唇 | mouthPressLeft/Right |
| AU24 | 嘴唇压住 | 压唇 | mouthPucker |
| AU25 | 嘴唇分开 | 张嘴 | jawOpen |
| AU26 | 下巴下垂 | 下巴下落 | jawOpen |

输出：

- 15 维 AU（与数据集严格对齐）：`AU1, AU2, AU4, AU6, AU7, AU9, AU10, AU12, AU14, AU15, AU17, AU23, AU24, AU25, AU26`
- 数值范围: [0, 1]，0表示无活动，1表示最大强度

备注：

- 可参考 PyLips 的开源映射思路，但以本项目映射表为准
- 必须与数据集 AU 种类一致对齐：最终输出的 AU 集合与命名（如 `AU01/AU02/...`）以数据集标注口径为准，禁止随意增删或改名，否则会导致训练/评测与线上推理无法对齐
- **对称化处理**: 左右AU取平均，如 `AU12 = (AU12_L + AU12_R) / 2`

### 4.3 头部姿态（pitch/yaw/roll）

技术路线：

- MediaPipe landmarks + solvePnP（PnP 解算）

输出：

- `pitch` / `yaw` / `roll`（单位：度）

备注：

- MediaPipe 侧可提供姿态相关矩阵/信息；若直接使用其输出，需统一坐标系定义

### 4.4 视线估计（可选）

技术选型：

- MediaPipe Face Landmarker（gaze 向量）

输出：

- 视线方向向量 `(x, y, z)`

备注：

- 可选，用于增强交互（例如对话时机、注意力检测）

### 4.5 输出帧率与实时性

- 目标：15~30 fps
- 建议：对下游输出限流到 15 fps 以节省 CPU

## 5. 处理流程与实现要点

### 5.1 异步推理（降低延迟）

- 使用 FaceLandmarker 的异步模式（`LIVE_STREAM`）
- 采用回调输出，避免阻塞 UI 线程/主循环
- **实现代码位置**: `services/vision/inference/mediapipe_face.py`

**v0.10.x API初始化示例**:

```python
import mediapipe as mp

# v0.10.x 新API
base_options = mp.tasks.BaseOptions(
    model_asset_path="d:/face_landmarker.task"  # 必须纯英文路径
)
options = mp.tasks.vision.FaceLandmarkerOptions(
    base_options=base_options,
    running_mode=mp.tasks.vision.RunningMode.LIVE_STREAM,
    output_face_blendshapes=True,
    output_facial_transformation_matrixes=True,
    num_faces=1,
    result_callback=_on_face_result  # 异步回调
)
landmarker = mp.tasks.vision.FaceLandmarker.create_from_options(options)

# 推流时传入帧
# v0.10.x: 使用 mp.Image 而非 mp.tasks.vision.MPImage
mp_image = mp.Image(
    image_format=mp.ImageFormat.SRGB,
    data=rgb_frame_array
)
landmarker.detect_async(mp_image, timestamp_ms=int(time.time() * 1000))
```

### 5.2 AU 映射、对称化与平滑

AU 映射：

- `au_k = Σ (w_i * blendshape_i)`，权重由 `NAME2AUWEIGHT` 提供
- **映射算法实现**: `services/vision/features/au_mapper.py`

对称化：

- 左右 AU 取平均（例如 AU_l 与 AU_r 合并为 AU_mean）
- **对称化规则**: 在 `au_mapping_v1.json` 中定义左右对应关系

平滑滤波（一阶 IIR）：

- `y_t = alpha * y_{t-1} + (1 - alpha) * x_t`
- 建议 `alpha = 0.7`（可配置）
- **平滑器实现**: `services/vision/features/smooth_filter.py`

### 5.3 从AU到情绪分类的映射规则（基线）

基于15维AU计算8类表情概率（规则基线，可后续用模型增强）：

```python
# 规则示例（需根据数据集校准）
expression_rules = {
    "Happy": 0.4 * AU12 + 0.3 * AU6 + 0.2 * AU1 + 0.1 * AU2,
    "Sad": 0.5 * AU15 + 0.3 * AU1 + 0.2 * AU4,
    "Anger": 0.4 * AU4 + 0.3 * AU7 + 0.2 * AU23 + 0.1 * AU24,
    "Surprise": 0.4 * AU1 + 0.4 * AU2 + 0.2 * AU26,
    "Fear": 0.3 * AU1 + 0.3 * AU2 + 0.2 * AU4 + 0.2 * AU20,
    "Disgust": 0.4 * AU9 + 0.3 * AU10 + 0.3 * AU17,
    "Contempt": 0.6 * AU12 + 0.4 * AU14,  # 单侧AU12
    "Neutral": 1 - max(other_expressions)
}
```

### 5.4 从AU到VA的映射规则（基线）

```python
# 基于AU的VA估计（需根据数据集校准）
valence = 0.5 * (AU12 - AU15) + 0.3 * (AU6 - AU4) + 0.2 * (AU1 + AU2)
arousal = 0.3 * (AU1 + AU2 + AU4) + 0.3 * (AU5 + AU7) + 0.2 * AU20 + 0.2 * AU26
```

### 5.3 输出形式（两种模式）

为兼顾“驱动精细度”和“链路负载”，模块支持两种输出：

- 模式 A（驱动优先）：输出 `blendshapes_52`（可选同时输出 landmarks）
- 模式 B（情感特征优先）：输出 `25` 维情感特征向量：`15 AU + 2 VA + 8 表情类别`

说明：

- VA 与 8 类表情可以来自：
  - AU/姿态/视线的规则融合（基线可用）
  - 轻量分类器（增强可选）

### 6.1 PerceptionToLLM.vision 字段约束

- `vision.enabled`：是否启用视觉；摄像头关闭/无权限/无脸时可为 `false`
- `vision.provider`：固定为 `mediapipe`
- `vision.model.name`：例如 `face_landmarker.task`
- `vision.mode`：`live_stream`
- `vision.frame_rate_fps`：实际输出帧率（建议 15）
- `vision.window_ms`：聚合窗口（建议 1000ms，用于把多帧汇总为本轮摘要）
- `vision.face_count`：本轮/窗口内有效人脸数（默认 1）
- `vision.processing.au_mapping.au_schema`：AU 对齐口径（必须）
  - `au_schema.source`：固定 `dataset`
  - `au_schema.schema_id`：数据集 AU 口径版本（例如 `dataset_au_v1`）
  - `au_schema.au_names`：本项目“线上输出”的 AU 名称列表，必须与数据集标注 AU 列表一致
  - `au_schema.va_names`：固定 `["valence","arousal"]`
  - `au_schema.expression_8_names`：固定 `["Neutral","Happy","Sad","Surprise","Fear","Disgust","Anger","Contempt"]`
  - 约束：输出的 AU 强度必须覆盖 `au_names` 中的全部 AU；若某 AU 当帧不可估计，必须输出 `intensity=0`（而不是缺失），避免下游维度漂移

### 6.4 数据集特征对齐（最终特征向量顺序）

当下游需要“25 维情感特征向量”时，必须按以下固定顺序拼接（与数据集列顺序一致）：

1. `AU1, AU2, AU4, AU6, AU7, AU9, AU10, AU12, AU14, AU15, AU17, AU23, AU24, AU25, AU26`
2. `valence, arousal`
3. `Neutral, Happy, Sad, Surprise, Fear, Disgust, Anger, Contempt`

### 6.3 兼容与负载建议

- `landmarks_478` 默认建议关闭，仅在调试/评测开启（payload 很大）
- 生产建议最小输出：`blendshapes_52` + `au_15` + `head_pose` + `summary`
- 必须在无脸时保持字段稳定：
  - `enabled=false`
  - `quality.tracking_state="no_face"`
  - `features` 可为空对象或省略

## 7. 质量与异常处理要求

### 7.1 人脸缺失/遮挡

- 无人脸：输出 `enabled=false`，并记录 `tracking_state="no_face"`
- 短暂丢失：可保持上一帧平滑后的状态不超过 `window_ms`，超过则置为无效

### 7.2 多人脸

默认策略：

- 选择“最大面积/最中心”的一张脸作为主脸输出

可选策略：

- 输出多脸列表（需要扩展 `features.face[]`，当前基线不启用）

### 7.3 坐标与单位

- landmarks 坐标：归一化坐标 `x/y/z`（由 MediaPipe 定义）
- head_pose：`pitch/yaw/roll` 单位为度
- gaze：方向向量建议归一化
- AU/blendshape：强度范围建议 `0~1`

## 8. 性能与资源约束

### 8.1 性能指标（建议基线）

| 指标 | 目标值 | 测试方法 |
|------|--------|----------|
| 输出帧率 | 15 fps（可配置到 30 fps） | 连续运行统计 |
| 单帧推理延迟 | ≤ 50ms | 单帧处理时间 |
| 端到端延迟 | ≤ 100ms（从采集到输出） | 包含平滑滤波 |
| CPU占用 | ≤ 30%（单核） | 任务管理器监控 |
| 内存占用 | ≤ 500MB | 包含模型加载 |
| 掉帧率 | ≤ 5% | `quality.dropped_frames` |

### 8.2 资源约束

- CPU 优先场景：强制 15 fps 限流 + 关闭 `landmarks_478`
- 若启用多任务（AU+姿态+视线）：优先复用 FaceLandmarker 的一次推理结果，避免重复计算

### 8.3 优化策略

1. **帧率自适应**: 根据CPU负载动态调整输出帧率（15-30fps）
2. **ROI裁剪**: 检测到人脸后，仅处理人脸区域附近（降低计算量）
3. **批量推理**: 支持多帧批量处理（离线评测场景）
4. **模型缓存**: FaceLandmarker实例复用，避免重复初始化
