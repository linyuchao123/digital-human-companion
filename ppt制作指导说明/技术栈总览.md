# 技术栈总览

## 项目名称
基于AI大语言模型的情感陪护虚拟数字人系统

---

## 一、后端与服务框架

| 技术 | 版本/说明 | 用途 |
|------|-----------|------|
| Python | 3.10 | 主要开发语言 |
| FastAPI | 0.100+ | REST API + WebSocket 后端框架 |
| Uvicorn | 0.23+ | ASGI 高性能服务器 |
| WebSocket | 原生支持 | 前后端实时双向通信 |
| asyncio | 标准库 | 异步并发处理 |
| ThreadPoolExecutor | 标准库 | CPU密集型任务线程池 |

---

## 二、大语言模型（LLM）

| 技术 | 说明 | 用途 |
|------|------|------|
| Qwen（通义千问） | API调用，支持Qwen-turbo/plus | 情感对话生成、心理疏导 |
| 联网搜索增强 | Qwen enable_search 参数 | RAG无结果时动态补充知识 |
| 上下文记忆 | 10轮+连续对话 | 个性化、持续性陪护 |

---

## 三、语音处理（ASR/TTS）

| 技术 | 版本/说明 | 用途 |
|------|-----------|------|
| FunASR | ModelScope离线部署 | 中文语音识别（ASR） |
| Paraformer-zh | FunASR核心模型 | 端到端语音转文字 |
| FSMN-VAD | FunASR配套 | 语音活动检测（VAD） |
| CT-Punc | FunASR配套 | 标点符号恢复 |
| CosyVoice | 阿里开源TTS | 语音合成，流式首包输出 |
| Web Speech API | 浏览器原生 | 前端实时语音输入 |

## 十、评测指标达成情况

| 指标 | 要求 | 实现情况 |
|------|------|---------|
| 语音识别词错率（WER） | ≤ 10% | FunASR Paraformer-zh 达标 |
| 语音识别句错率（SER） | ≤ 40% | LCS全文搜索算法优化后达标 |
| 连续对话轮次 | ≥ 10轮 | Qwen上下文窗口 + 会话记忆 |
| LLM上下文窗口 | ≥ 8K tokens | Qwen-turbo 8K+ |
| 端到端响应时间 | ≤ 60秒 | FastAPI异步化 + 流式输出 |
